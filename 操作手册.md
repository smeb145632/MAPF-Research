## ⚠️ 竞赛核心规则

### 🚫 绝对禁止

根据[Prepare_Your_Submission.md](./Prepare_Your_Submission.md)：
- **禁止修改 `src/`、`inc/` 目录下的任何文件**
- 只能修改和添加代码在 `default_planner/` 文件夹下
- 禁止修改编译相关文件（`compile.sh`、`CMakeLists.txt`）的核心逻辑
- 禁止删除任何源文件

### ✅ 唯一允许的操作区域

**`default_planner/`** 是唯一允许新增和修改代码的地方，包括：
- `default_planner/planner.cpp` / `planner.h` - 规划器实现
- `default_planner/scheduler.cpp` / `scheduler.h` - 调度器实现
- `default_planner/heuristics.cpp` / `heuristics.h` - 启发式函数
- `default_planner/const.h` - 常量参数

## 🔧 编译与运行

### 编译方式

```bash
# 在 WSL 环境下
cd /mnt/f/MAPF/MAPF-Research
./compile.sh
```

### 运行方式

```bash
./build/lifelong --inputFile <输入文件> -o <输出文件>
```

**示例**：

```bash
./build/lifelong --inputFile ./example_problems/random.domain/random_32_32_20_100.json -o test.json
```

### 常用命令行参数

| 参数 | 说明 |
|------|------|
| `--inputFile` / `-i` | 输入问题文件路径（必需） |
| `--output` / `-o` | 输出JSON文件路径 |
| `--simulationTime` / `-s` | 最大执行tick数 |
| `--preprocessTimeLimit` / `-p` | 预处理时间限制（毫秒） |
| `--initialPlanTimeLimit` / `-n` | 首次规划时间限制（毫秒） |
| `--planCommTimeLimit` / `-t` | 规划更新最小间隔（毫秒） |
| `--outputScreen` / `-c` | 输出详细程度：1=完整，2=摘要，3=仅统计 |

## 📋 Git 工作流程

### 分支策略

```
master（主分支，保护分支，禁止直接提交）
```

### 开发流程（必须遵循）

```
1. 📤 从 master 签出新分支
   git checkout master
   git pull origin master
   git checkout -b feature/你的功能名

2. ✏️ 在 default_planner/ 下修改或新增代码

3. 🧪 自测
   - 编译：./compile.sh
   - 运行测试用例
   - 检查 numPlannerErrors 和 numScheduleErrors

4. 🤔 反思
   - 代码是否有优化空间？
   - 是否引入新的问题？
   - 是否符合竞赛规范？

5. 🔀 合入 master
   git add .
   git commit -m "描述你的修改"
   git push origin feature/你的功能名
   # 提 PR/MR 到 master，经审核后合入

6. 🔄 切换回 master（合入后）
   git checkout master
   git pull origin master
```

### Git 命令速查

```bash
# 查看当前分支
git branch

# 查看状态
git status

# 查看修改
git diff

# 提交
git add .
git commit -m "fix: 修复了XXX问题"
```
